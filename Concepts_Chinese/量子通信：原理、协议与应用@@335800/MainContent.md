## 引言
几个世纪以来，保护我们的秘密一直是一场复杂性的较量，是制造锁的人与撬锁的人之间的竞赛。这就是经典密码学的世界，其安全性依赖于那些对于今天的技术而言实际上无法解决的计算难题。然而，这种安全性是有条件的，它持续受到计算技术进步的威胁，尤其是[量子计算](@article_id:303150)机的黎明。但是，如果我们能将安全性的基础不是建立在计算机的局限性上，而是建立在宇宙自身不可动摇的法则之上呢？

本文将探讨量子通信这一革命性领域，它恰好提供了这样的承诺：[信息论安全性](@article_id:300497)。它通过将安全性植根于量子力学原理中，解决了经典加密的根本漏洞。在阅读过程中，您将发现一个新世界，在这里，仅仅是窃听秘密对话这一行为本身，就会留下不可否认的证据。

首先，在**原理与机制**一章中，我们将深入探索使这一切成为可能的核心物理学。我们将探讨深刻的不可克隆定理，剖析优雅的[BB84协议](@article_id:299096)以理解密钥的诞生过程，并检验纠缠在验证安全性方面所扮演的令人费解的角色。然后，在**应用与跨学科联系**一章中，我们将看到这些抽象原理如何被工程化为现实世界的技术，解决古老的密码学难题，构筑未来[量子互联网](@article_id:303879)的骨干，并与其他科学学科建立起令人惊讶的联盟。

## 原理与机制

想象一下，您想发送一封秘密信息。几个世纪以来，您能做的最好的事情就是把它锁在一个有着非常复杂锁具的盒子里。希望在于没有其他人拥有钥匙，或者足够聪明能撬开这把锁。这就是经典[密码学](@article_id:299614)的世界。其安全性依赖于**计算困难性**——即假设某些数学问题，如分解巨大的数字，对于任何当前或可预见的计算机来说都过于困难。但如果有一天，有人造出了一台强大得多的计算机呢？例如，一台[量子计算](@article_id:303150)机就能以惊人的速度撬开许多这类现代锁具。因此，您信息的安全性是*有条件的*，它仅仅在您的锁比对手的聪明才智更难撬开时才有效。[@problem_id:1651408]

[量子通信](@article_id:299437)提供了一个全新的、激进的承诺：一种并非由人类智慧的局限性，而是由自然基本法则所保证的安全性。这就像拥有一个只在有人试图篡改时才会告诉您，且只告诉您的盒子。任何“撬锁”的尝试都不可避免地以一种可被探测到的方式破坏盒子。这被称为**[信息论安全性](@article_id:300497)**，它是安全通信的圣杯。让我们来一窥这个非凡盒子的内部。

### 你无法复制一声耳语：不可克隆原理

量子安全俱乐部的第一条规则是：你不能复制一个未知的[量子态](@article_id:306563)。这不是一个指导方针，而是一条宇宙法则，被称为**不可克隆定理**。想象你有一个粒子，比如一个[光子](@article_id:305617)，处于一个特定但未知的[量子态](@article_id:306563) $|\psi\rangle$。你想建造一台机器，它接收这个[光子](@article_id:305617)和一个“空白”[光子](@article_id:305617)，然后输出两个都处于 $|\psi\rangle$ 状态的[光子](@article_id:305617)。听起来像一台量子复印机，对吧？

量子力学的法则是不可动摇的：任何系统的演化都必须是一个*线性*变换。让我们看看这意味着什么。假设我们的机器可以完美复制一个水平[偏振光](@article_id:336856)子，将 $|H\rangle$ 变成 $|H\rangle \otimes |H\rangle$，以及一个[垂直偏振](@article_id:325169)[光子](@article_id:305617)，将 $|V\rangle$ 变成 $|V\rangle \otimes |V\rangle$。现在，如果我们给它输入一个*叠加态*，比如一个对角偏振光子 $|\psi\rangle = \frac{1}{\sqrt{2}}(|H\rangle + |V\rangle)$ 呢？

由于机器是线性的，它对和的作用必须是作用的和。所以，它会产生状态 $\frac{1}{\sqrt{2}}(|H\rangle \otimes |H\rangle + |V\rangle \otimes |V\rangle)$。但我们*想要*的是一个完美的副本，那应该是 $|\psi\rangle \otimes |\psi\rangle = \frac{1}{2}(|H\rangle \otimes |H\rangle + |H\rangle \otimes |V\rangle + |V\rangle \otimes |H\rangle + |V\rangle \otimes |V\rangle)$。这两个结果状态是根本不同的！机器失败了。事实上，可以证明没有线性机器能够对所有可能的输入状态完成这项工作。[@problem_id:2236806]

这一个原理就是[量子密码学](@article_id:305253)的基石。一个窃听者，我们称之为 Eve，不能简单地拦截我们发送方 Alice 的量子信息，复制一份，然后将原始信息发送给接收方 Bob。为了了解信息内容，Eve 被迫要去测量它。而在量子世界里，测量的行为本身就是一个侵入性的、改变游戏规则的事件。

### 问对问题：一个保密协议 (BB84)

最著名的[量子密钥分发](@article_id:298519)方案是 **BB84 协议**，以其发明者 Charles Bennett 和 Gilles Brassard 的名字命名。它完美地展示了如何将不可克隆原理转化为一个实用的安全工具。

#### 如何发送一个秘密

Alice 想发送一串随机比特（这将成为密钥）给 Bob。她一次发送一个比特，将每个比特编码到单个[光子](@article_id:305617)的偏振上。但诀窍在于：她使用两种不同的“语言”，或者说**基**，来编码她的比特。

1.  **直线基 (+)**：她将 '0' 编码为水平偏振 ($|H\rangle$)，'1' 编码为[垂直偏振](@article_id:325169) ($|V\rangle$)。
2.  **对角基 (×)**：她将 '0' 编码为 45° 对角偏振 ($|D\rangle$)，'1' 编码为 135° 反对角偏振 ($|A\rangle$)。

对于她发送的每个比特，Alice 随机抛硬币决定使用哪个基。在接收端的 Bob 也面临同样的情况。对于他接收到的每个[光子](@article_id:305617)，他不知道 Alice 使用了哪种语言。所以，他也抛硬币，并随机选择在直线基 (+) 或对角基 (×) 中进行测量。

在他们交换了一长串[光子](@article_id:305617)之后，他们通过一个公共[信道](@article_id:330097)——一个普通的电话[线或](@article_id:349408)互联网连接——做一些听起来很奇怪的事情：Alice 告诉 Bob 她为每个[光子](@article_id:305617)使用了哪个基，而 Bob 也告诉她他使用了哪个基。他们不透露比特值，只透露他们所问的*问题的类型*。

然后他们执行一个“**筛选**”过程。当他们的基选择碰巧匹配时，他们就保留这个比特。如果他们使用了不同的基（例如，Alice 用 '+' 基发送，而 Bob 用 '×' 基测量），结果是无意义的，所以他们丢弃那个比特。剩下的比特串，即“筛选后的密钥”，在理想世界中，应该是一个只有 Alice 和 Bob 知道的秘密字符串。

#### 窃听者的困境

现在，让我们把我们的间谍 Eve 加入进来。她守在[信道](@article_id:330097)上，拦截 Alice 发送的每一个[光子](@article_id:305617)。由于不可克隆定理，她不能只是复制[光子](@article_id:305617)。她必须测量它来获知比特值。但她应该使用哪个基呢？她和 Bob 处境相同；她必须猜测。

假设 Alice 想要发送一个 '0'，并选择了直线基，发送一个 $|H\rangle$ [光子](@article_id:305617)。[@problem_id:2236843]
- 如果 Eve 猜对了，也使用了直线基，她会测量到 $|H\rangle$，断定比特是 '0'，然后发送一个全新的 $|H\rangle$ [光子](@article_id:305617)给 Bob。如果 Bob 也碰巧选择了直线基，他会测量到 $|H\rangle$ 并正确地记录一个 '0'。没有引入错误。
- 但如果 Eve 猜*错*了，使用了对角基呢？一个 $|H\rangle$ 态是 $|D\rangle$ 和 $|A\rangle$ 的等量叠加。她的测量将以 50% 的概率随机得到 '0' (态 $|D\rangle$) 或 '1' (态 $|A\rangle$)。假设她测得了 $|D\rangle$。她现在发送一个处于 $|D\rangle$ 态的新[光子](@article_id:305617)给 Bob。现在，当 Bob（他应该与 Alice 的 '+' 基匹配）在直线基中测量这个 $|D\rangle$ [光子](@article_id:305617)时，他将有 50% 的概率得到 $|H\rangle$ ('0') 或 $|V\rangle$ ('1')。他有 50% 的机会得到错误的比特！

Eve 的干涉留下了痕迹。由于必须猜测基，她不可避免地在筛选后的密钥中引入了错误——即那些 Alice 和 Bob 都使用了相同基的比特。对于像这样的简单“截获重发”攻击，事实表明，Eve 的猜测游戏会在筛选后的密钥中引入 25% 的错误率，这个值被称为**[量子比特错误率](@article_id:304232)（QBER）**。[@problem_id:2236843] 令人惊奇的是，即使 Eve 对某个基有偏好，这个结论也成立。猜测的行为本身就出卖了她。[@problem_id:2236843] 同样的原理也适用于使用不同态集或基的其他协议；在“错误”基下的测量总是会冒着扰动状态并产生可探测错误的风险。[@problem_id:2111552] [@problem_id:1651411]

筛选之后，Alice 和 Bob 只需在公共[信道](@article_id:330097)上比较他们筛选后密钥的一小部分随机样本。如果他们发现的错误比自然噪声预期的要多，他们就知道有人在窃听。他们会丢弃密钥并重新开始。如果错误率很低，他们就可以确信[信道](@article_id:330097)是安全的。

### 一种更深的联系：通过纠缠实现安全 (E91)

还有另一种，或许更令人费解的方式来达到同样的目标，被称为 **E91 协议**。在这种协议中，不是由 Alice 制备和发送[光子](@article_id:305617)，而是一个源产生**纠缠**粒子对，并将其中一个发送给 Alice，另一个发送给 Bob。

纠缠是 Einstein 所说的“鬼魅般的超距作用”。想象两枚被魔法般连接起来的硬币。无论它们相隔多远，如果你抛起其中一枚，它正面朝上，你瞬间就知道另一枚在抛起时会反面朝上。量子版本要微妙和强大得多。Alice 和 Bob 收到的粒子，其测量结果以一种无法用经典物理解释的方式相关联。

为了检查 Eve 的存在，他们不仅仅是寻找比特错误。他们进行一个更深层次的检验——一个对实在性本身的检验。在他们[纠缠粒子](@article_id:314103)的一个随机子集上，他们各自从几个不同的测量设置中随机选择。之后，他们公开比较他们的设置和结果，以计算一个通常用 $S$ 表示的值，该值出现在**[贝尔不等式](@article_id:316645)**（如 CHSH 不等式）中。经典物理学和常识（一种被称为“[局域实在论](@article_id:305406)”的世界观）要求 $|S| \le 2$。而量子力学预测，对于[纠缠态](@article_id:303351)，这个不等式可以被违背；例如，$S$ 可以达到 $2\sqrt{2}$。

如果 Eve 试图拦截并测量这些粒子，她会破坏它们之间精巧的纠缠。Alice 和 Bob 随后观察到的关联将变得“经典”，他们计算出的 $S$ 值将服从 $|S| \le 2$ 的规则。所以，通过执行[贝尔检验](@article_id:307567)并发现像 $|S|=2.5$ 这样的结果，他们不仅展示了宇宙最深刻的特征之一，而且同时也在验证他们[信道](@article_id:330097)的绝对安全性！而一个诸如 $|S| = 1.60$ 的结果则会通不过检验，并发出警报，表明纠缠已被破坏，[信道](@article_id:330097)已受到危害。[@problem_id:1651392]

### 从噪声数据到黄金密钥：经典清理小组

现在，Alice 和 Bob 拥有了一个筛选后的密钥，并且他们相信没有人在*过多地*窃听。但他们的工作还没有完成。这个密钥仍然不完美。它包含来自噪声（以及任何残余窃听）的错误，并且 Eve 可能仍然掌握了关于它的部分信息。这时，一个纯粹的经典后处理阶段，就像一个数字清理小组，便登场了。

#### 计算伤疤：参数估计

首先，他们必须非常准确地估计 QBER。他们通过公开牺牲并比较他们筛选后密钥的一个随机样本来做到这一点。但这是一个统计测量。如果他们测试 50,000 个比特并发现 750 个错误，他们测得的 QBER 是 $1.5\%$。但整个密钥的*真实* QBER 可能会稍高一些。为了获得可证明的安全性，他们必须持悲观态度，并计算 QBER 的一个保守上限，同时考虑到统计涨落。这在真实世界的系统中是一个关键步骤，尤其是当密钥长度不是无限的时候。[@problem_id:2111547]

#### 达成一致：[信息协商](@article_id:305933)

接下来，他们必须使他们的密钥完全相同。这被称为**[信息协商](@article_id:305933)**或纠错。他们使用巧妙的[算法](@article_id:331821)，通过在公共[信道](@article_id:330097)上交换消息来找到并修复不匹配的比特。这个过程是一支精巧的舞蹈。他们需要透露刚好足够的信息来修复错误，但又要尽可能少，以避免给 Eve 太多关于密钥本身的线索。他们必须揭示的最小信息量由信息论决定，并通过**[二元熵函数](@article_id:332705)** $h_2(Q)$ 与 QBER 相关。这代表了达成一致的代价。[@problem_id:1651380]

#### 擦除间谍的笔记本：[隐私放大](@article_id:307584)

最后，他们来到了最后一个，几乎是神奇的步骤：**[隐私放大](@article_id:307584)**。此时，Alice 和 Bob 共享一个相同的比特串。然而，Eve 掌握了一些部分知识。她从监听协商过程中学到了一点，也从她可能在量子信道上进行的任何测量中学到了一点。QBER 是他们用来衡量 Eve 可能拥有的最大知识量的把手。

为了消除 Eve 的知识，他们执行一个最终操作。他们将他们长的、部分保密的密钥通过一种称为 2-通用[哈希函数](@article_id:640532)的特殊函数。这个函数接收一个长输入并产生一个较短的输出。结果是一个更短，但现在几乎完全随机和保密的最终密钥。他们需要缩短密钥的量是经过精确计算的，刚好足以使 Eve 关于最终密钥的信息变得可以忽略不计。

这正是著名的[密钥率](@article_id:305459)公式 $R \approx 1 - 2h_2(Q)$ 中第二个 $h_2(Q)$ 项的目的。在初始的原始信息（'1'）中，一部分 $h_2(Q)$ 用于纠错，另一部分，同样由 $h_2(Q)$ 估算，必须被牺牲掉以抹去 Eve 的知识。[@problem_id:1651398] 这两个步骤必须按正确的顺序进行：首先，你们就一个共享的秘密达成一致（协商），然后，你们提纯它的纯度（放大）。[@problem_id:1647747]

通过这个多阶段过程，从量子世界的奇异法则开始，到[经典信息论](@article_id:302461)的优雅[算法](@article_id:331821)结束，Alice 和 Bob 可以提炼出一个“黄金密钥”——一个相同的、随机的，并且对于拥有任何计算能力的对手而言，完全且可证明地保密的比特串。
