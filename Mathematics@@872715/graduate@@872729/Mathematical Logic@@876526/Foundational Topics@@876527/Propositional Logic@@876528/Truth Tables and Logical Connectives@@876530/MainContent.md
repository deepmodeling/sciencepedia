## Principles and Mechanisms

In the study of classical [propositional logic](@entry_id:143535), the semantic framework provides the foundation for assigning meaning—specifically, truth or falsity—to logical formulas. This chapter delves into the principles and mechanisms that govern this assignment, beginning with the core concept of truth-functionality and culminating in a [complete theory](@entry_id:155100) of expressive power. We will see how the simple yet powerful tool of the truth table arises from these principles, serving not only as a definitional device but also as a complete algorithm for logical decision-making.

### The Foundation: Truth-Functional Compositionality

The semantics of classical [propositional logic](@entry_id:143535) rests on the **Principle of Compositionality**, which asserts that the truth value of a complex formula is determined entirely by the [truth values](@entry_id:636547) of its constituent parts. The "meaning" of a logical connective, in this context, is precisely the function it computes on these [truth values](@entry_id:636547). This is why such connectives are termed **extensional** or, more commonly, **truth-functional**.

Formally, we begin with a set of [truth values](@entry_id:636547), which for classical logic is the two-element Boolean algebra $\{0, 1\}$, where $1$ represents truth and $0$ represents falsity. A **truth assignment** (or **valuation**) is a function $v: \mathsf{Var} \to \{0,1\}$ that assigns a truth value to each propositional variable in a given set of variables $\mathsf{Var}$. [@problem_id:2987709]

The essence of truth-functional semantics is that any such assignment $v$ can be extended in one and only one way to a valuation $\hat{v}: \mathsf{Fm} \to \{0,1\}$ defined on the entire set of [well-formed formulas](@entry_id:636348) $\mathsf{Fm}$. This extension is defined by [recursion](@entry_id:264696) on the structure of the formulas. For any given $n$-ary connective $\circ$, there is a fixed truth function $f_\circ: \{0,1\}^n \to \{0,1\}$ such that:
$$ \hat{v}(\circ(\varphi_1, \dots, \varphi_n)) = f_\circ(\hat{v}(\varphi_1), \dots, \hat{v}(\varphi_n)) $$

The [existence and uniqueness](@entry_id:263101) of this extension $\hat{v}$ is a cornerstone of [logical semantics](@entry_id:637245), guaranteed by the inductive structure of the set of formulas. There are two powerful perspectives from which to justify this guarantee [@problem_id:2987709]:

1.  **The Principle of Structural Recursion**: The set of formulas $\mathsf{Fm}$ is freely generated from the set of variables $\mathsf{Var}$ by the connectives. This means every formula has a unique "recipe" or [parse tree](@entry_id:273136). The Principle of Structural Recursion states that a function on such a set is uniquely determined once its values on the atomic elements (the variables) are specified, and a rule is given for computing its value on a complex element from its values on the immediate sub-elements. Our semantic definition does precisely this.

2.  **The Universal Property of Free Algebras**: From the perspective of universal algebra, $\mathsf{Fm}$ is the *absolutely free algebra* generated by $\mathsf{Var}$ over the signature of connectives. The set of [truth values](@entry_id:636547) $\{0,1\}$ can be made into an algebra of the same signature by interpreting each connective as its corresponding truth function (e.g., $\land$ is interpreted as the function $\min(x,y)$). The universal property of free algebras states that any function from the generators (our valuation $v$ on $\mathsf{Var}$) to another algebra (our truth-value algebra) extends to a *unique homomorphism* (our full valuation $\hat{v}$). A homomorphism is, by definition, a function that respects the algebraic structure—which is exactly what truth-functionality demands.

A direct and crucial consequence of this principle is that the truth value of a formula $\varphi$ under a valuation $v$ depends solely on the assignments $v$ makes to the variables that actually appear in $\varphi$, denoted $\mathrm{Var}(\varphi)$. Any two valuations $v$ and $w$ that agree on all variables in $\mathrm{Var}(\varphi)$ will necessarily yield the same truth value for $\varphi$, i.e., $v(\varphi) = w(\varphi)$. The value is independent of the syntactic form (e.g., $(p \vee q) \wedge r$ vs. $p \vee (q \wedge r)$), the formula's proof history in some deductive system, or the valuation's assignment to variables not present in the formula. [@problem_id:2987715]

### The Standard Connectives and the Material Conditional

The abstract principle of [compositionality](@entry_id:637804) is made concrete through the definitions of the standard [logical connectives](@entry_id:146395), which are specified by their [truth tables](@entry_id:145682). For propositional variables $A$ and $B$, the classical connectives are defined as follows, where $v(A)$ and $v(B)$ are the [truth values](@entry_id:636547) of $A$ and $B$:

-   **Negation ($\neg$)**: $v(\neg A) = 1 - v(A)$. It is true if and only if its input is false.
-   **Conjunction ($\land$)**: $v(A \land B) = \min\{v(A), v(B)\}$. It is true if and only if both inputs are true.
-   **Disjunction ($\lor$)**: $v(A \lor B) = \max\{v(A), v(B)\}$. It is true if and only if at least one input is true.
-   **Biconditional ($\leftrightarrow$)**: $v(A \leftrightarrow B) = 1$ if $v(A) = v(B)$, and $0$ otherwise. It is true if and only if the inputs have the same truth value.

The **[material conditional](@entry_id:152262)** ($A \to B$) is perhaps the most debated of the classical connectives, particularly its truth when the antecedent $A$ is false. Its definition can be rigorously motivated by first principles. The primary desideratum for a conditional is that it must license the rule of inference known as **Modus Ponens**: from $A$ and $A \to B$, we must be able to infer $B$. Semantically, this means that in any valuation where both $A$ and $A \to B$ are true, $B$ must also be true. Formally, for any valuation $v$, if $v(A)=1$ and $v(A \to B)=1$, it must be that $v(B)=1$. [@problem_id:2987733]

Let us analyze this constraint on the four possible input pairs for the truth function $f_{\to}(v(A), v(B))$:
- If $v(A)=1$ and $v(B)=1$: The constraint is $\min(1, f_{\to}(1,1)) \le 1$, which is always true.
- If $v(A)=0$ and $v(B)=1$: The constraint is $\min(0, f_{\to}(0,1)) \le 1$, which is always true.
- If $v(A)=0$ and $v(B)=0$: The constraint is $\min(0, f_{\to}(0,0)) \le 0$, which is always true.
- If $v(A)=1$ and $v(B)=0$: The constraint is $\min(1, f_{\to}(1,0)) \le 0$. This forces $f_{\to}(1,0) = 0$.

Modus Ponens thus fixes only one entry of the [truth table](@entry_id:169787): a true antecedent cannot lead to a false consequent in a true conditional. To fix the remaining three entries, we can invoke a principle of **maximal truth** (or minimal commitment): a statement is held to be true unless it is forced to be false. Applying this principle, we assign the value $1$ to all entries not constrained to be $0$. This yields the standard truth table for the [material conditional](@entry_id:152262):

$$
\begin{array}{cc|c}
A  B  A \to B \\
\hline
0  0  1 \\
0  1  1 \\
1  0  0 \\
1  1  1
\end{array}
$$

This table corresponds to the formula $\neg A \lor B$. This principle-based derivation demonstrates that the [material conditional](@entry_id:152262) is not an arbitrary definition but the most permissive binary connective that respects Modus Ponens. [@problem_id:2987733]

### Truth Tables as a Decision Procedure

A **truth table** for a formula $\varphi$ is a tabular representation that enumerates all possible [truth assignments](@entry_id:273237) to its constituent variables and computes the resulting truth value of $\varphi$ for each assignment. For a formula containing $n$ distinct propositional variables, there are $2^n$ unique [truth assignments](@entry_id:273237). This is because a truth assignment is a function from the set of $n$ variables to the 2-element set of [truth values](@entry_id:636547), and there are $|T|^{|V|} = 2^n$ such functions. Each row of the [truth table](@entry_id:169787) corresponds to one of these unique assignments. [@problem_id:2987696]

By constructing the full [truth table](@entry_id:169787) for a formula, we can classify its semantic status:
- A formula is a **[tautology](@entry_id:143929)** if it evaluates to $1$ (true) in every row of its truth table.
- A formula is a **contradiction** if it evaluates to $0$ (false) in every row.
- A formula is **contingent** if it evaluates to $1$ in some rows and $0$ in others.

The truth table method provides a **decision procedure**: a mechanical, algorithmic process that is guaranteed to terminate and correctly determine whether a formula is a tautology, contradiction, or contingent. This procedure is **sound** (it never gives a wrong answer) and **complete** (it gives an answer for any formula) because it is a direct, brute-force implementation of the semantic definitions. It exhaustively checks every possible valuation. [@problem_id:2987695]

This method can be extended to determine the validity of a logical argument. An argument with premises $\Gamma = \{\psi_1, \dots, \psi_k\}$ and conclusion $\varphi$ is semantically valid if $\Gamma$ **semantically entails** $\varphi$, written $\Gamma \vDash \varphi$. This is defined to mean that for every valuation $v$, if $v(\psi_i)=1$ for all $\psi_i \in \Gamma$, then $v(\varphi)=1$. In other words, there is no valuation that makes all premises true and the conclusion false.

To check for entailment using a truth table, one constructs the table for all premises and the conclusion. Then, one identifies all rows in which every premise is true. If the conclusion is also true in all of those rows, the entailment holds. As an example, we can verify the validity of Modus Ponens, $\{A \to B, A\} \vDash B$:

| $v(A)$ | $v(B)$ | $v(A \to B)$ | Premises True? | Conclusion True? |
| :---: | :---: | :---: | :---: | :---: |
| $0$ | $0$ | $1$ | No ($v(A)=0$) | |
| $0$ | $1$ | $1$ | No ($v(A)=0$) | |
| $1$ | $0$ | $0$ | No ($v(A \to B)=0$) | |
| $1$ | $1$ | $1$ | Yes | Yes |

The only row where both premises are true is the last one, and in that row, the conclusion $B$ is also true. Thus, the entailment is valid. [@problem_id:2987707]

### Expressive Power and Functional Completeness

While we have focused on a standard set of connectives, any function $f: \{0,1\}^n \to \{0,1\}$ defines a possible logical connective. This raises a natural question: can our standard set of connectives express all other possible connectives? A set of connectives is said to be **functionally complete** if any arbitrary Boolean function can be represented by a formula using only those connectives.

The set $\{\neg, \land, \lor\}$ is functionally complete. However, smaller sets can also be complete. Remarkably, there exist single binary connectives that are functionally complete on their own. The two most famous examples are the **Sheffer Stroke** (NAND, written $A \mid B$) and the **Peirce Arrow** (NOR, written $A \downarrow B$). [@problem_id:2987732]

-   **NAND ($A \mid B$)**: Defined as $\neg(A \land B)$, it is false if and only if both $A$ and $B$ are true. It is functionally complete because we can define a known complete set, such as $\{\neg, \lor\}$:
    -   $\neg A \equiv A \mid A$
    -   $A \lor B \equiv (A \mid A) \mid (B \mid B)$

-   **NOR ($A \downarrow B$)**: Defined as $\neg(A \lor B)$, it is true if and only if both $A$ and $B$ are false. It is functionally complete because we can define a known complete set, such as $\{\neg, \land\}$:
    -   $\neg A \equiv A \downarrow A$
    -   $A \land B \equiv (A \downarrow A) \downarrow (B \downarrow B)$

The existence of such connectives is not merely a curiosity; it has profound implications for the design of [digital logic circuits](@entry_id:748425), where NAND or NOR gates can be used as universal building blocks for any computational circuit.

### A Deeper Theory of Expressiveness: Post's Criterion

The question of [functional completeness](@entry_id:138720) can be answered systematically and completely through a beautiful result known as Post's Completeness Theorem. The theory revolves around the idea of a **clone**, which is a set of Boolean functions that is closed under composition and contains all projection functions (e.g., $\pi_1(x, y) = x$). A function $f$ is definable from a set of connectives $C$ if and only if $f$ belongs to the clone generated by $C$. [@problem_id:2987730]

A set of functions $C$ cannot be functionally complete if all functions in $C$ share a certain property that is preserved under composition, and this property is not shared by all Boolean functions. Emil Post identified five such "invariant" properties, which define five maximal subclones of the set of all Boolean functions. A set of connectives $C$ is functionally complete if and only if it is not entirely contained within any of these five classes. [@problem_id:2987716]

The five maximal clones (also known as Post's classes) are:
1.  The set $T_0$ of **falsity-preserving** functions: those for which $f(0, 0, \dots, 0) = 0$.
2.  The set $T_1$ of **truth-preserving** functions: those for which $f(1, 1, \dots, 1) = 1$.
3.  The set $M$ of **monotone** functions: those for which an increase in the inputs (changing a $0$ to a $1$) never causes the output to decrease (change from $1$ to $0$). For instance, the clone generated by the ternary [majority function](@entry_id:267740) $M(x,y,z)$ and the constant functions $0$ and $1$ is precisely the set of all [monotone functions](@entry_id:159142). [@problem_id:2987730]
4.  The set $S$ of **self-dual** functions: those for which $f(x_1, \dots, x_n) = \neg f(\neg x_1, \dots, \neg x_n)$.
5.  The set $L$ of **affine** (or linear) functions: those that can be expressed as $f(x_1, \dots, x_n) = a_0 \oplus a_1 x_1 \oplus \dots \oplus a_n x_n$, where $\oplus$ is addition modulo $2$ (XOR).

**Post's Criterion** states that a set of connectives $C$ is functionally complete if and only if, for each of these five properties, $C$ contains at least one connective that *does not* have that property. This provides a complete algorithm for deciding [functional completeness](@entry_id:138720): for each of the five properties, check if there is a violator in the given set $C$. If a violator is found for all five, the set is complete. Each of these properties can be checked algorithmically by inspecting the [truth tables](@entry_id:145682) of the functions in $C$. [@problem_id:2987716]

### Beyond Extensionality: Intensional Connectives

The entire framework of this chapter has been built on the principle of extensionality, where connectives are truth-functional. This design has profound epistemic benefits: it guarantees that the truth value of a complex proposition can be computed mechanically and efficiently (in time linear in the size of the formula) given a valuation. It also guarantees a decision procedure for [logical validity](@entry_id:156732), albeit one with a complexity exponential in the number of variables. [@problem_id:2987698]

However, many expressions in natural language and other logical systems are **intensional**—their truth value depends on more than just the [truth values](@entry_id:636547) of their components. A primary example is the modal operator for necessity, $\Box$. In the standard Kripke semantics for [modal logic](@entry_id:149086), the truth of $\Box\varphi$ at a possible world $w$ is defined as follows: $\mathcal{M},w \vDash \Box\varphi$ if and only if $\mathcal{M},w' \vDash \varphi$ for all worlds $w'$ accessible from $w$.

The value of $\Box\varphi$ at $w$ is not determined by the value of $\varphi$ at $w$. To see this, consider a model with two worlds, $w$ and $u$, where $w$ can access $u$. Let's consider two different scenarios for a proposition $p$. In the first, $p$ is true at both $w$ and $u$. In the second, $p$ is true at $w$ but false at $u$. In both scenarios, the "truth value" of $p$ at the point of evaluation, $w$, is the same (true). However, $\Box p$ is true in the first scenario (since $p$ is true at all accessible worlds, namely $u$) but false in the second (since $p$ is false at the accessible world $u$). [@problem_id:2987698] This demonstrates that $\Box$ is not truth-functional; no finite truth table can capture its meaning. Its semantics are intensional, requiring an evaluation across a space of possibilities.

The study of [truth tables](@entry_id:145682) and truth-functional connectives thus defines the landscape of classical [propositional logic](@entry_id:143535). It is a world of perfect compositional transparency, algorithmic decidability, and syntactic simplicity. Understanding this world and its boundaries is the first and most critical step in appreciating the richer, more complex semantic structures that lie beyond.
