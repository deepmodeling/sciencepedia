**SciencePedia** 是一个**开放、协作、可信的科学知识库**，致力于构建人类的“科学知识宇宙”。我们以“学科”为脉络，以“知识点”为最小单元，追求科学知识的**可获取、结构化、可验证与动态更新**。

---

## 🌟 项目介绍

人类正处在一个**前所未有的科学信息爆炸“奇点”**，人工智能的到来，正在深刻重塑人类的学习、认知与创新范式。我们希望：

- 🛡️ **用严谨、开放与共识守护人类的科学积累**
- 🧭 **打造“科学知识导航”，帮助探索者定位自我、理解全貌**
- 📍 **生成“学习路径图谱”，指引每个人抵达未来的前沿**

为此，我们正式发起 **SciencePedia科学知识库开源计划**！

### 🚀 我们的目标
- ✅ 确定科学知识的**共识边界、结构图谱与验证机制**
- ✅ 支撑**智能化与个性化的科研与教育新范式**
- ✅ 加速**AI for Science涌现自主科研创新与科学发现**

### 🌈 我们的愿景
在不远的将来：
- 每个学科，都将拥有**完整、可验证的知识体系**
- 每个人，都能**免费获取准确、系统的科学知识**
- 每位学习者，都能**按需生成自己的学科体系与学习路径**
- 每个科研者，都能**自定义构建专属“科学外脑”**
- 每一项科学前沿进展，都能**自主汇入学科的演化脉络**

采用**最严格的知识验证与开放协作机制**：

| 机制 | 描述 |
|------|------|

## 🔧 **快速开始**

| **场景** | **操作步骤** | **示例效果** |

## 🤝 如何贡献

### 📱 **方式1：访客模式**
适用于所有人

| 步骤 | 操作 | 详情 |
|------|------|------|
| **1. Fork仓库** | 点击右上角Fork | 复制到你的账户 |
| **2. 编辑文件** | 点击铅笔图标 | 直接在网页修改 |
| **3. 提交更改** | 填写Commit消息 | 创建新分支自动提交 |
| **4. 创建PR** | Pull requests → New PR | 选择原仓库main分支 |
| **5. 等待合并** | 维护者审查 | 收到通知即成功！ |

### 🔧 **方式2：开发者模式**
适用于**Collaborator/Owner**

| 步骤 | 操作 | 详情 |
|------|------|------|
| **1. 新建分支** | 分支下拉 → Create branch | 如`feature-update` |
| **2. 编辑文件** | 点击铅笔图标 | 直接提交到新分支 |
| **3. 创建PR** | Pull requests → New PR | base:main, compare:你的分支 |
| **4. 合并PR** | Merge pull request | 选择Merge commit |

**最佳实践**：PR描述说明变更原因。
