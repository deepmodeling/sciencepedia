## 引言
[网络化控制系统](@entry_id:271631)（NCS）通过通信网络连接空间分布的传感器、控制器和执行器，已成为现代工程系统的基石，从[工业自动化](@entry_id:276005)到无人驾驶，其应用无处不在。这种[分布](@entry_id:182848)式架构在带来灵活性和[可扩展性](@entry_id:636611)的同时，也引入了严峻的挑战：通信网络并非理想的媒介。数据在传输过程中不可避免地会经历时延和[数据包丢失](@entry_id:269936)（简称“[丢包](@entry_id:269936)”），这些网络缺陷会严重降低控制性能，甚至破坏整个系统的稳定性。因此，理解、分析并克服这些网络非理想因素的影响，是设计可靠、高效NCS的核心问题。

本文旨在系统性地解决这一知识鸿沟。我们将带领读者深入探索时延与[丢包](@entry_id:269936)对控制系统的影响，并学习如何设计能够主动应对这些挑战的控制策略。通过本文的学习，你将能够：
- 在第一章“原理与机制”中，掌握为时延和[丢包](@entry_id:269936)建立精确数学模型的方法，并学习分析由此产生的时延系统和随机系统的稳定性，理解其背后的基本性能限制。
- 在第二章“应用与交叉学科联系”中，看到这些理论如何在[稳定性分析](@entry_id:144077)、[控制器设计](@entry_id:274982)以及与通信、信息论等学科的[交叉](@entry_id:147634)领域中得到实际应用。
- 在第三章“动手实践”中，通过具体问题加深对核心概念的理解和应用能力。

本章将为你奠定坚实的基础，引领你进入[网络化控制系统](@entry_id:271631)分析与设计的核心地带，首先从理解其背后的基本原理与机制开始。

## 原理与机制

在上一章引言的基础上，本章深入探讨构成[网络化控制系统](@entry_id:271631)（NCS）核心挑战的原理和机制。我们将系统地分析网络引入的非理想因素（主要是时延和[丢包](@entry_id:269936)）如何影响控制系统的行为，并建立用于分析和设计的数学模型。我们的目标是为理解这些影响并最终设计出能够容忍这些不确定性的[鲁棒控制](@entry_id:260994)策略奠定坚实的基础。

### 建模网络缺陷

对[网络化控制系统](@entry_id:271631)进行严谨分析的第一步是建立能够精确描述网络通信信道中非理想行为的数学模型。这些模型是后续稳定性分析和[控制器设计](@entry_id:274982)的基础。

#### [网络化控制系统](@entry_id:271631)的基本构成

一个典型的[网络化控制系统](@entry_id:271631)由一个连续时间的被控对象、一个传感器、一个控制器和一个执行器组成，它们通过通信网络相互连接。为了精确描述信息流，我们将通信路径分为两个独立的信道：**传感器-控制器（S-C）信道**和**控制器-执行器（C-A）信道**。每个信道都可能引入时延和[数据包丢失](@entry_id:269936)（即[丢包](@entry_id:269936)）。

为了处理这些网络不确定性，必须采用严谨的协议来确保系统的**因果性**——即在任何物理时刻 $t$ 的决策和操作只能依赖于到该时刻为止已经物理上接收到的信息。为了实现这一点，数据包通常包含**时间戳（time-stamps）**。例如，在时刻 $t_k$ 采样的测量值 $y_k$ 会被打包并附上时间戳 $t_k$。当控制器在稍后的时刻 $t_k^a$ 收到这个数据包时，它不仅知道测量值 $y_k$，还通过时间戳知道这个信息是何时生成的。这对于[状态估计](@entry_id:169668)和补偿时延至关重要。

在执行器端，由于控制指令数据包的到达时间不确定，通常会采用**保持器（hold device）**，例如**[零阶保持器](@entry_id:264751)（Zero-Order Hold, ZOH）**。当一个新数据包到达时，执行器更新其输出；在两个数据包到达之间，它会保持上一个指令值不变。这种机制确保了被控对象在任何时候都有一个确定的输入。当数据包由于时变时延而**[乱序](@entry_id:147540)到达（out-of-order arrivals）**时，时间戳变得尤为重要。控制器和执行器都必须采用单调应用规则：只处理或应用那些时间戳比当前已处理或已应用的时间戳更新的数据包，并丢弃任何过时的数据包。这些机制共同确保了即使在存在时延和[丢包](@entry_id:269936)的情况下，系统的整体运行也严格遵守因果性原则 [@problem_id:2726955]。

#### 时延的数学模型

时延是NCS中最普遍的非理想因素之一。根据[系统建模](@entry_id:197208)的需要，我们可以从不同角度对其进行数学描述。

首先，考虑一个具有执行时延 $\tau_u$ 和测量时延 $\tau_m$ 的[连续时间LTI系统](@entry_id:267448)。假设控制器是数字的，采用采样周期为 $h$ 的零阶保持策略，即在 $[t_k, t_{k+1})$ 区间内，控制器输出恒定的指令 $u_k$。由于执行时延 $\tau_u$（假设 $0  \tau_u  h$），被控对象在时间区间 $[t_k, t_k+\tau_u)$ 内实际接收到的输入是上一个周期的指令 $u_{k-1}$，而在 $[t_k+\tau_u, t_{k+1})$ 区间内才接收到当前周期的指令 $u_k$。通过求解状态方程，我们可以得到系统的精确离散时间模型：

$x_{k+1} = \Phi x_k + \Gamma_0 u_k + \Gamma_1 u_{k-1}$

其中，$x_k \triangleq x(t_k)$ 是在采样时刻的状态。矩阵 $\Phi$, $\Gamma_0$ 和 $\Gamma_1$ 是通过对[连续时间系统](@entry_id:276553)在一个[采样周期](@entry_id:265475)内的演化进行积分得到的。具体而言：

$\Phi = \exp(Ah)$

$\Gamma_0 = \int_{0}^{h-\tau_u} \exp(As) ds \cdot B$

$\Gamma_1 = \int_{h-\tau_u}^{h} \exp(As) ds \cdot B$

如果矩阵 $A$ 可逆，这些积分可以被解析地计算出来，例如 $\Gamma_0 = A^{-1}(\exp(A(h-\tau_u)) - I)B$ [@problem_id:2726996]。这个模型清晰地表明，即使是很小的、小于一个[采样周期](@entry_id:265475)的时延，也会在离散时间模型中引入复杂性，使得下一时刻的状态不仅依赖于当前输入 $u_k$，还依赖于过去的输入 $u_{k-1}$。

当网络时延可以被建模为[采样周期](@entry_id:265475)的整数倍，即一个已知的常数延迟 $d$ 时，一个强大的技术是**[状态增广](@entry_id:140869)（state augmentation）**。考虑一个具有输入时延的[离散时间系统](@entry_id:263935)：

$x_{k+1} = A x_k + B u_{k-d}$

为了应用标准的无时延[控制器设计](@entry_id:274982)方法，我们希望将这个系统转换为一个等价的无时延[LTI系统](@entry_id:271946)。这可以通过将“传输中”的输入（即已经发出但尚未作用于系统的输入）包含到[状态向量](@entry_id:154607)中来实现。我们定义一个增广状态向量 $x^{\text{aug}}_k$：

$x^{\text{aug}}_k = \begin{pmatrix} x_k \\ u_{k-d} \\ u_{k-d+1} \\ \vdots \\ u_{k-1} \end{pmatrix}$

这个增广[状态向量](@entry_id:154607)包含了预测未来状态所需的全部信息：系统的当前状态 $x_k$ 和信道中的状态（即队列中的输入 $u_{k-d}, \dots, u_{k-1}$）。通过推导 $x^{\text{aug}}_{k+1}$ 中每个分量的演化，我们可以得到一个标准的无时延[状态空间](@entry_id:177074)形式：

$x^{\text{aug}}_{k+1} = A_{\text{aug}} x^{\text{aug}}_{k} + B_{\text{aug}} u_k$

其中，增广[系统矩阵](@entry_id:172230)为：

$A_{\text{aug}} = \begin{pmatrix}
A   B   0   \cdots   0 \\
0   0   I_m   \cdots   0 \\
\vdots   \vdots   \ddots   \ddots   \vdots \\
0   0   \cdots   0   I_m \\
0   0   \cdots   0   0
\end{pmatrix}, \quad
B_{\text{aug}} = \begin{pmatrix}
0 \\
0 \\
\vdots \\
0 \\
I_m
\end{pmatrix}$

这种变换将时延系统的分析和设计问题转化为了一个更高维度的标准[LTI系统](@entry_id:271946)问题，极大地扩展了可用工具的范围 [@problem_id:2726982]。

#### [丢包](@entry_id:269936)的数学模型

与时延一样，[丢包](@entry_id:269936)也必须被精确地建模。一个简单而有效的方法是使用**随机[指示变量](@entry_id:266428)（indicator variables）**。

**伯努利模型（[独立同分布](@entry_id:169067)[丢包](@entry_id:269936)）**

最简单的[丢包](@entry_id:269936)模型是**伯努利模型**，它假设每个数据包的丢失是独立于其他数据包的，并且丢失的概率是恒定的。例如，我们可以定义一个伯努利[随机变量](@entry_id:195330) $\gamma_k^m \in \{0, 1\}$ 来描述S-C信道在时刻 $k$ 的状态，其中 $\gamma_k^m=1$ 表示数据包成功送达，$\gamma_k^m=0$ 表示[数据包丢失](@entry_id:269936)。

在[丢包](@entry_id:269936)发生时，系统必须有一个预设的策略。一个常见的策略是“保持上一个采样值”（hold-last-sample）。例如，在控制器端，如果在时刻 $k$ 期望的测量数据包 $y_{k-d_m}$ 未能到达（即 $\gamma_{k-d_m}^m = 0$），控制器就继续使用它在上一个时刻所使用的测量值 $\tilde{y}_{k-1}$。如果数据包成功到达，控制器则更新其测量值为新接收到的 $y_{k-d_m}$。这种行为可以用一个简洁的[递归公式](@entry_id:160630)来描述：

$\tilde{y}_k = \gamma_{k-d_m}^m y_{k-d_m} + (1 - \gamma_{k-d_m}^m) \tilde{y}_{k-1}$

同样，在执行器端，施加到被控对象上的实际输入 $\tilde{u}_k$ 也可以用类似的方式建模：

$\tilde{u}_k = \gamma_{k-d_u}^u u_{k-d_u}^c + (1 - \gamma_{k-d_u}^u) \tilde{u}_{k-1}$

其中 $u_{k-d_u}^c$ 是控制器在时刻 $k-d_u$ 计算出的指令。这些递归表达式为包含随机[丢包](@entry_id:269936)的NCS闭环系统提供了精确的数学描述 [@problem_id:2726997]。

**吉尔伯特-艾略特模型（突发性[丢包](@entry_id:269936)）**

在实际网络中，[丢包](@entry_id:269936)往往不是独立的，而是以“突发”（bursts）的形式出现，即连续多个[数据包丢失](@entry_id:269936)。**吉尔伯特-艾略特（Gilbert-Elliott）模型**通过一个双状态的马尔可夫链（Markov chain）来描述这种相关性。该链有两个状态：$D$（Delivered，成功送达）和$L$（Lost，丢失）。从一个状态到另一个状态的转移概率决定了[丢包](@entry_id:269936)的“记忆性”。例如，从$D$到$L$的概率 $\alpha$ 较小，而从$L$回到$D$的概率 $\beta$ 也较小，则系统倾向于长时间停留在成功传输状态或连续[丢包](@entry_id:269936)状态。

该马尔可夫链的单步[转移矩阵](@entry_id:145510) $P$ 可以写作：

$P = \begin{pmatrix} 1-\alpha  \alpha \\ \beta  1-\beta \end{pmatrix}$

当一个NCS的[丢包](@entry_id:269936)过程由这样的[马尔可夫链](@entry_id:150828)驱动时，其闭环系统就成为一个**马尔可夫跳变线性系统（Markov Jump Linear System, MJLS）**。系统的动态矩阵会根据[马尔可夫链](@entry_id:150828)的状态[随机切换](@entry_id:197998)。例如，如果采用[状态反馈控制器](@entry_id:203349) $u_k = Kx_k$，并在[丢包](@entry_id:269936)时采用零输入策略，则[闭环系统](@entry_id:270770)可以表示为 $x_{k+1} = A_{\theta_k} x_k$，其中 $\theta_k \in \{D, L\}$ 是马尔可夫链的状态。对应的[系统矩阵](@entry_id:172230)为：

$A_D = A + BK$ （当 $\theta_k = D$ 时，数据包送达）

$A_L = A$ （当 $\theta_k = L$ 时，[数据包丢失](@entry_id:269936)）

这个MJLS框架是分析具有相关性[丢包](@entry_id:269936)的NCS稳定性的标准工具。该[马尔可夫链](@entry_id:150828)存在一个[平稳分布](@entry_id:194199) $(\pi_D, \pi_L)$，其中平稳[丢包](@entry_id:269936)概率 $\pi_L$ 可以解析地求出，为 $\pi_L = \frac{\alpha}{\alpha+\beta}$ [@problem_id:2726956]。

### [网络化控制系统](@entry_id:271631)的[稳定性分析](@entry_id:144077)

在建立了网络缺陷的数学模型之后，我们转向核心问题：分析这些缺陷如何影响[闭环系统](@entry_id:270770)的稳定性。

#### 时延[系统稳定性](@entry_id:273248)

对于包含时延的系统，稳定性的概念分为两大类：**时延无关稳定性（delay-independent stability）**和**时延相关稳定性（delay-dependent stability）**。

一个[稳定性判据](@entry_id:755304)如果是**时延无关的**，那么它给出的条件能够保证系统对于任意大小的非负时延 $h \ge 0$ 都是稳定的。这类判据通常比较保守，但提供了最强的鲁棒性保证。例如，对于一阶[时滞系统](@entry_id:262890) $\dot{x}(t) = -\alpha x(t) - \beta x(t-h)$，我们可以使用Lyapunov-Razumikhin方法来推导时延无关稳定条件。选择[Lyapunov函数](@entry_id:273986) $V(x) = \frac{1}{2}x^2$，其导数为 $\dot{V}(t) = -\alpha x(t)^2 - \beta x(t)x(t-h)$。在Razumikhin条件 $|x(t-h)| \le |x(t)|$ 下，我们得到 $\dot{V}(t) \le -(\alpha - |\beta|)x(t)^2$。为了保证 $\dot{V}(t)$ 负定，必须有 $\alpha - |\beta| > 0$，即 $|\beta|  \alpha$。这个条件只与系统参数 $\alpha$ 和 $\beta$ 有关，与时延 $h$ 无关。

然而，许多系统在时延较小时是稳定的，但随着时延增大，系统会失稳。**时延相关稳定性**分析旨在找出系统保持稳定的最大时延，即**时延[裕度](@entry_id:274835)（delay margin）** $h^\star$。这类分析得到的条件明确依赖于时延 $h$，因此通常没有时延无关判据那么保守。一种标准的时延[相关分析](@entry_id:265289)方法是[频域](@entry_id:160070)法。通过分析系统的[特征方程](@entry_id:265849)，我们寻找使特征根穿越虚轴的临界时延。对于上述[一阶系统](@entry_id:147467)，[特征方程](@entry_id:265849)为 $s + \alpha + \beta e^{-sh} = 0$。稳定性边界出现在当 $s=i\omega$ 时，其中 $\omega$ 是穿越频率。通过求解这个复数方程，我们可以找到使系统恰好处于失稳边缘的 $(\omega, h)$ 对。例如，对于参数 $\alpha=1$ 和 $\beta=1.2$ 的系统，时延[无关条件](@entry_id:165299) $|1.2|  1$ 不满足，表明系统并非对所有时延都稳定。通过时延[相关分析](@entry_id:265289)，可以精确计算出其时延[裕度](@entry_id:274835)约为 $h^\star = 3.853$ 秒 [@problem_id:2726930]。这个例子清晰地展示了两种分析方法的区别和联系。

#### 随机系统的稳定性

当系统中存在随机[丢包](@entry_id:269936)时，系统的状态 $x_k$ 成为一个[随机过程](@entry_id:159502)。此时，[确定性系统](@entry_id:174558)中简单的稳定/不稳定二元概念不再适用，我们必须采用概率论的语言来描述其行为。主要有以下几种[随机稳定性](@entry_id:196796)概念：

1.  **依概率稳定（Stability in Probability）**：这是一种较弱的稳定性概念，要求状态轨迹以高概率停留在原点附近的小邻域内。
2.  **均方稳定（Mean-Square Stability, MSS）**：指状态的二阶矩（或能量）随时间收敛到零，即 $\lim_{k \to \infty} \mathbb{E}[\|x_k\|^2] = 0$。这是一个在工程中广泛应用的强稳定性概念，因为它保证了状态的平均能量会耗散。
3.  **几乎必然稳定（Almost Sure Stability, a.s. Stability）**：这是最符合直觉的稳定性概念，指系统状态的每一个样本[轨道](@entry_id:137151)（除了一个概率为零的集合外）都收敛到零，即 $\mathbb{P}(\lim_{k \to \infty} x_k = 0) = 1$。

这些稳定性概念之间存在一定的层次关系。对于[线性随机系统](@entry_id:184741)，**均方稳定是一个比几乎必然稳定更强的条件**。也就是说，如果一个系统是均方稳定的，它必然也是[几乎必然](@entry_id:262518)稳定的。然而，反之不成立。一个系统可能几乎所有[轨道](@entry_id:137151)都收敛到零，但由于存在极小概率的、能量极大的“逃逸”[轨道](@entry_id:137151)，导致其二阶矩发散，从而不是均方稳定的 [@problem_id:2726983]。

在标量系统 $x_{k+1} = a_k x_k$ 中，其中 $a_k$ 是独立同分布的[随机变量](@entry_id:195330)，这两个稳定性的判据有明确的数学形式：
- **均方稳定**当且仅当 $\mathbb{E}[a_k^2]  1$。
- **几乎必然指数稳定**当且仅当 $\mathbb{E}[\ln|a_k|]  0$。

通过[Jensen不等式](@entry_id:144269)可知 $\ln(\mathbb{E}[a_k^2]) \ge \mathbb{E}[\ln(a_k^2)] = 2\mathbb{E}[\ln|a_k|]$。因此，$\mathbb{E}[a_k^2]  1$ 意味着 $\ln(\mathbb{E}[a_k^2])  0$，从而 $2\mathbb{E}[\ln|a_k|]  0$，再次印证了MSS比a.s.稳定性更强 [@problem_id:2726983]。

分析这些[随机稳定性](@entry_id:196796)的一个核心工具是**[随机Lyapunov理论](@entry_id:192740)**。其基本思想是考察Lyapunov函数 $V(x_k)$ 的[条件期望](@entry_id:159140)的单步变化。一个关键的充分条件是：如果存在一个正定函数 $V(x)$ 和一个常数 $c_3 > 0$，使得

$\mathbb{E}[V(x_{k+1}) | \mathcal{F}_k] - V(x_k) \le -c_3 \|x_k\|^2$

几乎必然成立（其中 $\mathcal{F}_k$ 是到时刻 $k$ 为止的系统历史信息），则系统是均方[渐近稳定](@entry_id:168077)的。这个条件直观地表示，在给定当前状态下，Lyapunov函数的[期望值](@entry_id:153208)在下一步会严格减小 [@problem_id:2726990]。

### 基本限制与设计原则

网络的存在不仅增加了分析的复杂性，更对可实现的控制性能施加了根本性的限制。

#### [均方稳定性](@entry_id:165904)分析

将NCS建模为马尔可夫跳变线性系统 $x_{k+1} = A_{\theta_k} x_k$ 后，我们可以分析其[均方稳定性](@entry_id:165904)。这通常归结为分析一个确定性[线性算子](@entry_id:149003)的稳定性。考虑系统状态的二阶矩矩阵 $X_k = \mathbb{E}[x_k x_k^T]$。其演化规律为：

$X_{k+1} = \mathbb{E}[A_{\theta_k} x_k x_k^T A_{\theta_k}^T] = \mathbb{E}[A_{\theta_k} X_k A_{\theta_k}^T]$

其中最后一个等号成立是因为在i.i.d.[丢包](@entry_id:269936)模型下，$\theta_k$ 与 $x_k$ 相互独立。这个递推关系描述了一个作用在[对称矩阵](@entry_id:143130)空间上的线性算子。系统的[均方稳定性](@entry_id:165904)等价于该[算子的谱半径](@entry_id:261858)小于1。使用[向量化算子](@entry_id:183061)和克罗内克积，这个条件可以表示为 $\rho(\mathbb{E}[A_\theta \otimes A_\theta])  1$ [@problem_id:2726983]。对于[独立同分布](@entry_id:169067)的伯努利[丢包](@entry_id:269936)，如果[丢包](@entry_id:269936)率为 $p$，则这个期望可以具体计算为：

$\mathbb{E}[A_\theta \otimes A_\theta] = p(A \otimes A) + (1-p)((A+BKC) \otimes (A+BKC))$

通过计算该矩阵的谱半径，就可以判断系统在给定控制器 $K$ 和[丢包](@entry_id:269936)率 $p$ 下是否均方稳定 [@problem_id:2726988]。

#### [丢包](@entry_id:269936)施加的基本限制

一个深刻的结果是，即使控制器的能力（即控制增益 $K$）没有限制，网络[丢包](@entry_id:269936)率本身也可能导致系统无法稳定。考虑一个标量不稳定开环系统 $x_{k+1} = ax_k + bu_k$（其中 $|a|>1, b \neq 0$），在带有[丢包](@entry_id:269936)率 $p$ 的执行通道下使用[状态反馈](@entry_id:151441) $u_k = Kx_k$。

系统的均方稳定条件是 $\mathbb{E}[(a+b\gamma_k K)^2]  1$，其中 $\gamma_k$ 是伯努利[指示变量](@entry_id:266428)，$\gamma_k=0$表示[丢包](@entry_id:269936)。这个条件可以展开为 $a^2 p + (a+bK)^2(1-p)  1$。这是一个关于 $K$ 的二次不等式。为了使系统**可镇定（stabilizable）**，我们只需要存在一个 $K$ 使得该不等式成立。这等价于该二次函数关于 $K$ 的最小值小于1。通过求导，可以找到最优增益 $K_{opt} = -a/b$，它使得二次函数达到最小值 $a^2p$。

因此，均方可镇定的充要条件是 $a^2 p  1$，即 $p  1/a^2$。这意味着存在一个**临界[丢包](@entry_id:269936)率** $p_{\text{crit}} = 1/a^2$。如果实际的[丢包](@entry_id:269936)率 $p$ 超过这个阈值，那么无论[控制器设计](@entry_id:274982)得多么“激进”，都无法实现均方稳定。这个结果揭示了一个根本性的权衡：被控对象越不稳定（即 $|a|$ 越大），系统对网络[丢包](@entry_id:269936)的容忍度就越低（即 $p_{\text{crit}}$ 越小）[@problem_id:2726967]。

#### 数据率施加的基本限制

当通信信道不仅有[丢包](@entry_id:269936)，还有**有限的数据率（data rate）**时，另一个基本限制浮出水面。这引出了控制理论与信息论的[交叉](@entry_id:147634)领域。稳定一个不[稳定系统](@entry_id:180404)的过程，可以看作是控制器通过有限的信道容量，与系统状态不确定性的增长进行对抗。

一个不稳定的线性系统 $x_{k+1} = Ax_k + \dots$，其[状态空间](@entry_id:177074)体积沿着不稳定特征方向会按 $|\lambda_i|$（其中 $|\lambda_i| \ge 1$）的速率扩张。从信息论的角度看，要描述一个不断膨胀的[随机变量](@entry_id:195330)（即系统状态）而不让不确定性无限增长，需要不断地向系统注入信息。被控对象状态不确定性的增长速率（或说熵的产生速率）正比于所有不稳定模式扩张率的对数和：

$R_{\text{plant}} = \sum_{|\lambda_i| \ge 1} \log_2 |\lambda_i|$ （单位：比特/步）

另一方面，一个容量为 $C$ 比特/包、[丢包](@entry_id:269936)率为 $p$ 的信道，其平均能够可靠传输的信息速率为：

$R_{\text{ch}} = (1-p) C$ （单位：比特/步）

为了实现均方稳定，信息注入的速率必须大于不确定性增长的速率。这导致了著名的**数据率定理**：对于一个通过有损、有限容量信道控制的[线性时不变系统](@entry_id:276591)，其可实现均方镇定的一个必要条件是：

$(1-p)C > \sum_{|\lambda_i| \ge 1} \log_2 |\lambda_i|$

这个定理优雅地将系统动力学特性（由 $A$ 的不稳定[特征值](@entry_id:154894)决定）、网络[服务质量](@entry_id:753918)（由[丢包](@entry_id:269936)率 $p$ 决定）和信道物理限制（由容量 $C$ 决定）联系在了一起，为[网络化控制系统](@entry_id:271631)的设计提供了根本性的指导原则。在某些假设下（例如信道有确认机制），这个条件也被证明是充分的 [@problem_id:2727013]。

本章通过对时延和[丢包](@entry_id:269936)进行系统性的建模与分析，揭示了它们对[控制系统稳定性](@entry_id:170483)的深刻影响，并阐述了由此产生的基本性能限制。这些原理与机制是设计和评估高级[网络化控制](@entry_id:168556)策略的基石。
